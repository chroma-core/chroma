Write path:
  add       - top-level add(id, embedding): navigate + register + balance
  navigate  - query centroid HNSW index for nearest clusters
  register  - quantize vector and append to target cluster(s)
  spawn     - create a new cluster and insert centroid into HNSW indexes
Balance path:
  scrub     - load cluster from reader, remove tombstoned entries
  split     - split an oversized cluster via 2-means into two new clusters
  merge     - merge an undersized cluster into its nearest neighbor
  reassign  - re-add a displaced vector after split/merge
  drop      - remove a cluster from centroid indexes and load its raw embeddings
I/O:
  load      - load quantized cluster data from blockfile reader into memory
  load_raw  - load raw f32 embeddings from blockfile reader into cache
Quantization:
  quantize  - encode a vector into a RaBitQ code relative to its centroid
Search:
  search    - full query: centroid navigate + cluster scan + rerank
  search_scan   - time in cluster scan (quantized distance over codes)
  search_rerank - time in data rerank (full-precision distance after load_raw)
Derived:
  raw_pts   - number of raw embedding points loaded during load_raw
  raw/pt    - average time per raw embedding point (load_raw total / raw_pts)
  total     - wall-clock time for the checkpoint (load + raw_write + index + commit)
  rr_vecs   - avg number of full-precision vectors loaded per search (data rerank)
  rr_data   - avg data volume of full-precision vectors per search (vecs x dims x 4B)
