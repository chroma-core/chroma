# Chroma History Search - Browser Extension

A Chrome extension that semantically searches your browser history using Chroma compiled to WebAssembly. The Chroma vector store runs natively in your browser via WASM - no server, no API keys, complete privacy.

## Architecture

```
extension/
├── manifest.json          # Chrome Manifest V3 config
├── background.js          # Service worker: indexes history, handles search
├── popup.html/css/js      # Search UI popup
├── lib/
│   ├── embeddings.js      # WASM embedding via Transformers.js (ONNX Runtime)
│   └── vector-store.js    # Chroma WASM wrapper with IndexedDB persistence
└── pkg/                   # Built WASM output (from rust/wasm/)
    ├── chroma_wasm.js     # JS bindings generated by wasm-pack
    └── chroma_wasm_bg.wasm # Compiled Chroma WASM binary

rust/wasm/                 # Chroma WASM source (in repo root)
├── Cargo.toml
└── src/
    ├── lib.rs             # wasm-bindgen bindings (ChromaCollection API)
    ├── collection.rs      # Pure Rust in-memory collection with brute-force search
    └── distance.rs        # Scalar distance functions (cosine, L2, inner product)
```

### How It Works

1. **Vector Store**: Chroma's collection and distance functions are compiled from Rust to WebAssembly via `wasm-pack`. The `ChromaCollection` WASM module provides `add()`, `query()`, `delete()`, `save()`, and `load()` methods - the same Chroma API, running natively in the browser.

2. **Embedding**: The [all-MiniLM-L6-v2](https://huggingface.co/Xenova/all-MiniLM-L6-v2) sentence transformer runs in ONNX Runtime WASM via Transformers.js (~23MB, downloaded once and cached).

3. **Indexing**: On install, the background service worker reads Chrome history via the `chrome.history` API, generates embeddings, and stores everything in the Chroma WASM collection.

4. **Persistence**: The Chroma collection is serialized to JSON and persisted in IndexedDB. On restart, the collection is restored from the snapshot.

5. **Search**: Query text is embedded using the same model, then searched against the Chroma collection using cosine similarity - all in WASM.

6. **Updates**: An alarm triggers incremental re-indexing every hour.

## Building the WASM Module

Prerequisites:
- [Rust](https://rustup.rs/) toolchain
- [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/)

```bash
# Install wasm-pack if you don't have it
curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh

# Build the Chroma WASM module
cd rust/wasm
wasm-pack build --target web --out-dir ../../examples/browser_history_search/extension/pkg

# The output goes directly into the extension's pkg/ directory
```

The build produces:
- `pkg/chroma_wasm.js` - JavaScript bindings with `ChromaCollection` class
- `pkg/chroma_wasm_bg.wasm` - Compiled WebAssembly binary (~50KB gzipped)

## Installation

1. Build the WASM module (see above)

2. Open Chrome and navigate to `chrome://extensions/`

3. Enable **Developer mode** (toggle in the top right)

4. Click **Load unpacked** and select this `extension/` directory

5. The extension icon will appear in your toolbar

6. Click the icon to open the search popup

## First Run

On first install, the extension will:

1. Load the Chroma WASM module (~50KB)
2. Download the embedding model (~23MB, cached after first download)
3. Read your last 90 days of browser history
4. Generate embeddings and store them in the Chroma collection

Progress is shown in the popup status bar.

## Usage

1. Click the extension icon in your toolbar
2. Type a natural language query, e.g.:
   - "that article about React hooks"
   - "python documentation"
   - "recipe for chocolate cake"
   - "stackoverflow git rebase"
3. Optionally filter by domain (e.g., "github.com")
4. Click a result to open the page

Search happens as you type (after a 500ms pause) or when you press Enter.

## Chroma WASM API

The `ChromaCollection` WASM class exposes the familiar Chroma interface:

```js
import init, { ChromaCollection } from './pkg/chroma_wasm.js';
await init();

// Create a collection
const collection = new ChromaCollection("my_collection", "cosine");

// Add entries with embeddings and metadata
collection.add(
  ["id1", "id2"],                           // IDs
  [[0.1, 0.2, ...], [0.3, 0.4, ...]],      // Embeddings
  ["document 1", "document 2"],              // Documents
  [{ key: "value" }, { key: "value2" }],     // Metadata
);

// Semantic search
const results = collection.query(
  [0.1, 0.2, ...],   // Query embedding
  10,                  // Number of results
  { key: "value" },   // Optional metadata filter
);
// results = [{ id, distance, document, metadata }, ...]

// Persist to string (store in IndexedDB/localStorage)
const snapshot = collection.save();

// Restore from string
const restored = ChromaCollection.load(snapshot);
```

## Permissions

- **history**: Read browser history for indexing
- **storage**: Store extension settings
- **alarms**: Schedule periodic re-indexing

## Privacy

- Chroma runs entirely as WASM in your browser - no server calls
- The embedding model is downloaded from HuggingFace CDN and cached locally
- No browsing data is sent to any server
- Collection data is stored in your browser's IndexedDB
- Uninstalling the extension removes all stored data
